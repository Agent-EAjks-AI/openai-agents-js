当您[运行智能体](/openai-agents-js/zh/guides/running-agents)时，您将会收到：

- 如果在调用 `run` 时未设置 `stream: true`，则为[`RunResult`](/openai-agents-js/openai/agents/classes/runresult)
- 如果在调用 `run` 时设置了 `stream: true`，则为[`StreamedRunResult`](/openai-agents-js/openai/agents/classes/streamedrunresult)。有关流式传输的详细信息，请参阅[流式传输](/openai-agents-js/zh/guides/streaming)。

## 最终输出

`finalOutput` 属性包含最后运行的智能体的最终输出。该结果可能为：

- `string` — 任何未定义 `outputType` 的智能体的默认输出
- `unknown` — 如果智能体的输出类型定义为 JSON 模式。在这种情况下 JSON 已解析，但仍需您手动校验其类型。
- `z.infer<outputType>` — 如果智能体的输出类型定义为 Zod 模式。输出将自动按照该模式解析。
- `undefined` — 如果智能体未产生输出（例如在产生输出之前已停止）

如果您使用了具有不同输出类型的交接，应该使用 `Agent.create()` 方法而不是 `new Agent()` 构造函数来创建智能体。

这将使 SDK 能够推断所有可能交接路径上的输出类型，并为 `finalOutput` 属性提供联合类型。

## 下一轮输入

有两种方式可获取下一轮的输入：

- `result.history` — 包含您的输入以及智能体输出的副本。
- `result.output` — 包含整次智能体运行的输出。

## 最后一个智能体

`lastAgent` 属性包含最后运行的智能体。根据您的应用，这通常对用户下次输入很有用。比如，如果您有一个前线分诊智能体会交接到特定语言的智能体，您可以存储上次的智能体，并在用户下次发消息时复用它。

在流式模式下，访问映射到当前运行智能体的 `currentAgent` 属性也很有用。

## 新项

`newItems` 属性包含在运行期间生成的新项。这些项是[`RunItem`](/openai-agents-js/openai/agents/type-aliases/runitem)。运行项包装了由 LLM 生成的原始项。除了访问 LLM 的输出外，还可以用它们来判断这些事件与哪个智能体相关。

- [`RunMessageOutputItem`](/openai-agents-js/openai/agents/classes/runmessageoutputitem) 表示来自 LLM 的消息。原始项为生成的消息。
- [`RunHandoffCallItem`](/openai-agents-js/openai/agents/classes/runhandoffcallitem) 表示 LLM 调用了交接工具。原始项为来自 LLM 的工具调用项。
- [`RunHandoffOutputItem`](/openai-agents-js/openai/agents/classes/runhandoffoutputitem) 表示发生了交接。原始项为对交接工具调用的工具响应。您也可以从该项访问源/目标智能体。
- [`RunToolCallItem`](/openai-agents-js/openai/agents/classes/runtoolcallitem) 表示 LLM 调用了某个工具。
- [`RunToolCallOutputItem`](/openai-agents-js/openai/agents/classes/runtoolcalloutputitem) 表示某个工具被调用。原始项为工具响应。您也可以从该项访问工具输出。
- [`RunReasoningItem`](/openai-agents-js/openai/agents/classes/runreasoningitem) 表示来自 LLM 的推理项。原始项为生成的推理。
- [`RunToolApprovalItem`](/openai-agents-js/openai/agents/classes/runtoolapprovalitem) 表示 LLM 请求批准某个工具调用。原始项为来自 LLM 的工具调用项。

## 状态

`state` 属性包含本次运行的状态。大多数附加在 `result` 上的信息都源自 `state`，但 `state` 可序列化/反序列化，并且在您需要[从错误中恢复](/openai-agents-js/zh/guides/running-agents#exceptions)或处理[`中断`](#interruptions)时，也可作为后续调用 `run` 的输入。

## 中断

如果您在智能体中使用了 `needsApproval`，您的 `run` 可能会触发一些需要在继续之前处理的 `interruptions`。在这种情况下，`interruptions` 将是导致中断的 `ToolApprovalItem` 数组。有关如何处理中断的更多信息，请参阅[人机协作](/openai-agents-js/zh/guides/human-in-the-loop)。

## 其他信息

### 原始响应

`rawResponses` 属性包含智能体运行期间由模型生成的原始 LLM 响应。

### 最后响应 ID

`lastResponseId` 属性包含智能体运行期间由模型生成的最后一个响应的 ID。

### 护栏结果

`inputGuardrailResults` 和 `outputGuardrailResults` 属性包含护栏的结果（如果有）。护栏结果有时包含您想记录或存储的有用信息，因此我们将其提供给您。

### 原始输入

`input` 属性包含您提供给 run 方法的原始输入。在大多数情况下您不会需要它，但在需要时也可用。
