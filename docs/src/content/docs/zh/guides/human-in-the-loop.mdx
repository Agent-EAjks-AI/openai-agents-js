本指南演示如何使用 SDK 中内置的人工干预支持，根据人工介入来暂停和恢复智能体运行。

目前的主要用例是对敏感的工具执行请求审批。

### 流程

1. 当智能体决定调用某个（或多个）工具时，会通过评估 `needsApproval` 检查该工具是否需要审批。
2. 如果需要审批，智能体会检查审批是否已被授予或拒绝。
   - 如果审批尚未授予或拒绝，工具会向智能体返回一条静态消息，说明该工具调用无法执行。
   - 如果审批/拒绝缺失，将触发一个工具审批请求。
3. 智能体会收集所有工具审批请求并中断执行。
4. 如果发生任何中断，[执行结果](/openai-agents-js/zh/guides/results) 中会包含一个描述待处理步骤的 `interruptions` 数组。当某个工具调用需要确认时，会出现一个 `type: "tool_approval_item"` 的 `ToolApprovalItem`。
5. 您可以调用 `result.state.approve(interruption)` 或 `result.state.reject(interruption)` 来批准或拒绝该工具调用。
6. 处理完所有中断后，您可以将 `result.state` 传回 `runner.run(agent, state)`（其中 `agent` 是触发整体运行的原始智能体）以恢复执行。
7. 流程将从第 1 步重新开始。

## 处理较长的审批时间

人工干预流程被设计为可在较长时间内中断而无需保持您的服务器运行。如果您需要中止请求并稍后继续，可以序列化状态并在之后恢复。

您可以使用 `JSON.stringify(result.state)` 序列化状态，之后将序列化的状态传入 `RunState.fromString(agent, serializedState)` 以恢复（其中 `agent` 是触发整体运行的智能体实例）。

这样，您就可以将序列化的状态存储在数据库中，或与您的请求一起存放。

### 待处理任务的版本控制

<Aside>
  这主要适用于在对智能体进行变更的同时，需要将序列化状态长时间存储的情形。
</Aside>

如果审批请求需要较长时间，并且您打算以有意义的方式为智能体定义进行版本化，或升级您的 Agents SDK 版本，我们目前建议您使用包别名并行安装两个版本的 Agents SDK，自行实现分支逻辑。

实践中，这意味着为您自己的代码分配一个版本号，将其与序列化状态一同存储，并在反序列化时引导到正确的代码版本。
